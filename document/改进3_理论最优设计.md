# 改进3 理论最优设计：L1 换 L2

**日期**: 2026-07-01
**前提**: 以下分析抛开 epoch 数量，纯从理论基础推导最优方案。

---

## 〇、架构事实

一个关键但被忽略的事实：

```
ic_head_1(feat_spic_1) → C₁ ← 仅作为 loss 辅助输出，不进 ICPG
ic_head_2(feat_spic_2) → C₂ ← 仅作为 loss 辅助输出，不进 ICPG
ic_enc_3(feat_spic_2) → ic_map_3(=C₃) → ic_enc_4 → ic_feature → ICPG
```

C₁/C₂ 是纯辅助输出，不参与前向数据流。进入 ICPG 的只有从 ic_enc_3→ic_enc_4 提取的 ic_feature（单通道）。一致性约束的实际作用是**正则化 ic_enc_1 和 ic_enc_2 的中间特征**，使它们在 ICSP 链中传递时不会让下游 ic_enc_3 收到矛盾信息。

## 一、一致性到底该度量什么：空间排序，不是精确匹配

三层复杂度头有不同的等效感受野：

| 层 | 感受野 | 物理含义 |
|------|:---:|------|
| ic_head_1 | ~21×21 | 局部纹理边界强度 |
| ic_head_2 | ~45×45 | 中尺度类别混杂度 |
| ic_head_3 | ~69×69 | 大尺度语义难度 |

三层看到的物理世界不同——它们**本就应该输出不同的复杂度图**。强制 C₁=C₃ 会摧毁多尺度互补性。正确的约束目标应该是：**对"哪个区域更难"的空间排序达成一致**——允许量级差异，但 pattern 对齐。

| 度量 | 允许量级差异？ | 允许局部合理差异？ | 梯度自适应？ |
|------|:---:|:---:|:---:|
| L1 | ✗ | 部分 | ✗ |
| **L2 (MSE)** | **✓** | **✓** | **✓** |
| Pearson 相关 | ✓✓ | ✗ (全局) | ✓ |

Pearson 理论上更纯（完全量级无关），但数值不稳定（平坦图方差为零时 NaN），梯度全局耦合。L2 是最优权衡。

## 二、为什么 L1 在数学上就是错的

考虑 C₁ 的联合优化：

$$\nabla L_{total}(C_1) = w_1 \cdot \nabla L_{IC}(C_1, label) + \lambda_{cons} \cdot \nabla d(C_1, C_3)$$

| | ∇d(C₁, C₃) | 行为 |
|------|------|------|
| L1 | λ × sign(C₁−C₃) / N | 恒定幅值，永不退场 |
| L2 | 2λ × (C₁−C₃) / N | 幅值与偏差成正比 |

### 稳态时（|C₁−C₃|≈0.01）

- L2 梯度 ≈ 2×10⁻⁸，监督项主导 → C₁/C₂ 自由适应自己的感受野 ✅
- L1 梯度 ≈ 5×10⁻⁷，持续压制 → 过正则化 ❌

### 偏离时（C₃ 发生表征变化，|C₁−C₃|≈0.2）

- L2 梯度自动涨至 4×10⁻⁷，及时拉回 ✅
- L1 梯度仍是 5×10⁻⁷，不会加速 ❌

**一个好的正则项应该：对齐时退场（让数据说话），偏离时介入（拉回来）。** L1 做不到第一点，所以是坏的正则项。

## 三、优化理论：光滑 vs 非光滑

L_total 在 L1 下于 C₁=C₃ 处**不可微**——目标函数存在尖点。在此点梯度不唯一定义，子梯度方法的收敛速率从线性退化至次线性。在 SGD 训练中，非光滑性表现为持续的梯度噪声：每次 iter 的更新方向在尖点附近来回震荡，形成隐式的优化阻力。

L2 下目标函数全局光滑。光滑目标的梯度下降在最优值附近具有线性收敛速率。

## 四、贝叶斯视角：先验分布的选择

```
L1 对应 Laplace 先验:  P(C₁|C₃) ∝ exp(−λ|C₁−C₃|)
L2 对应 Gaussian 先验: P(C₁|C₃) ∝ exp(−λ(C₁−C₃)²)
```

Laplace 先验认为 C₁ 以极高概率精确等于 C₃，偶尔出现大幅偏离。这描述的物理场景是"C₁ 就是 C₃"，但这与事实矛盾——C₁ 和 C₃ 有不同的感受野，它们**本来就该不同**。C₁ 在边界附近的高激活响应是合理的多尺度行为，不应被惩罚。

Gaussian 先验认为 C₁ 围绕 C₃ 分布，小偏差概率高，大偏差概率低。这精确描述了"C₁ 大致跟随 C₃ 的趋势，但允许在合理范围内保持独立表达"——与多尺度表征的物理事实一致。

## 五、多任务学习视角：损失权重比

IC KD 对 C₁ 的监督损失（含层权重 w₁=0.15）：

$$L_{IC\_1} = w_1 \times k_{ic} \times (k_{mse} \times \text{MSE} + k_{kd} \times \tau^2 \times \text{KL})$$

在 mid-training 状态（IC loss ≈ 0.51, |C₁−C₃| ≈ 0.1）：

| 方案 | 一致性损失 | 监督损失 | 比率 |
|------|:---:|:---:|:---:|
| L1, λ=0.5 | 0.05 | 0.018 | **2.8:1** 🔴 |
| L2, λ=0.5 | 0.005 | 0.018 | **0.28:1** ✅ |

一致性是正则项，不应该是主目标。2.8:1 的比例完全倒挂了主次。L2 通过损失量的自然衰减，自动将一致性置于辅助位置。

而且 L2 下的比例是自调节的：

| 状态 | L2 一致性 | 监督 | 比率 |
|------|:---:|:---:|:---:|
| 对齐良好 (|C₁−C₃|=0.05) | 0.0013 | 0.018 | 1:14 |
| 正常 (|C₁−C₃|=0.1) | 0.005 | 0.018 | 1:3.6 |
| 偏离 (|C₁−C₃|=0.2) | 0.02 | 0.018 | 1.1:1 |

不需要调 λ_cons 来适配不同训练阶段——L2 自动做了这件事。

## 六、BYOL 视角：detach 必须保留

BYOL 的核心发现：对称一致性导致表示坍塌（所有输出退化至常数）。打破对称性（stop_gradient + 非对称 predictor）是防止坍塌的必要条件。

我们的 `detach(C₃)` 等价于 BYOL 的 stop_gradient。如果移除 detach 变成双向约束：
- C₃ 的梯度既受伪标签驱动（IC loss）又受 C₁/C₂ 拉动（一致性 loss）
- 两者矛盾时 C₃ 被"稀释"
- 最坏情况：三层收敛至输出相同的中间值，多尺度多样性被摧毁

detach(C₃) 保证 C₃ 始终以伪标签为唯一优化目标，不受浅层干扰。

## 七、最终方案

```python
# 改动：F.l1_loss → F.mse_loss，其余不变
ic_cons_loss = self.k_ic_cons * (
    F.mse_loss(C1, C3.detach()) + F.mse_loss(C2, C3.detach())
)
```

| 设计选择 | 结论 | 理由 |
|------|:---:|------|
| L1 → L2 | **必须改** | L1 恒定梯度是结构性缺陷，不可通过调参修复 |
| detach(C₃) | **保留** | BYOL 证明非对称性是防止坍塌的必要条件 |
| λ_cons | **0.5 可保留** | L2 损失量自体衰减，对 λ 不敏感 |
| 方向 | **单向，非双向** | 深层锚点不应被浅层拉偏 |
| 需要 ramp-up？ | **不需要** | L2 的梯度自适应等于隐式 ramp-up——早期偏差大时梯度自动强，对齐后自动弱 |
| 需要归一化？ | **不需要** | C₁/C₂/C₃ 已 Sigmoid 至 [0,1] |

## 八、改进1+2+3 实验数据的回顾

L1 (λ=0.5) 的 50 epoch 实验给出了 56.50% mIoU（+3.85 vs 1+2）。这个结果已经不错——说明一致性约束的方向是对的。但 L1 的局限性可能限制了它的潜力。换成 L2 后，预期至少不低于 56.50%，且收敛曲线应更平滑（无尖锐平台期）。

## 九、总结

**一行代码的改动，从五个理论框架（梯度分析、光滑优化、贝叶斯先验、多任务学习、BYOL）各自独立导出了同一结论：L1 → L2。**

这不是调参，是修 bug。
